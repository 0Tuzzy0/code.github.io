Unit teszt példa ChatGPT segitségével:

// sum.js

function add(a, b) {
return a + b;
}
module.exports = add;

Most készítsünk egy egyszerű Jest tesztet ennek a függvénynek:

// sum.test.js

const add = require('./sum');
test('adds 1 + 2 to equal 3', () => {
expect(add(1, 2)).toBe(3)
});
test('adds 5 + 7 to equal 12', () => {
expect(add(5, 7)).toBe(12);
});

Ebben a példában a 'sum.test.js' fájlban két egyszerű tesztesetet hoztunk létre a
'add' függvényhez. Az első teszteset ellenőrzi, hogy az 'add(1, 2)' hívás eredménye
megegyezik-e a várt értékkel (3), míg a második teszteset az 'add(5, 7)' hívás
eredményét ellenőrzi a várt értékkel (12).

A 'test' függvények a Jest keretrendszer részei, és az 'expect' függvény
segítségével definiálják a várt eredményt. Ha a függvény valóban megfelel a
várakozásoknak, akkor a teszt sikeres lesz, különben hibát jelez

Ez csak egy egyszerű példa, de a valóságban a unit tesztek általában sokkal
összetettebbek és több különböző esetet fednek le. A unit tesztek segítenek abban,
hogy a kódunk stabil legyen, és könnyebben karbantartható legyen.
